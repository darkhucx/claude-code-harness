### 33.1 PreToolUse additionalContext 活用 `cc:完了` `[feature:high-impact]`

> **v2.1.9 新機能**: PreToolUse フックが `additionalContext` を返してモデルに追加情報を注入可能

**現状**:
- `pretooluse-guard` は `continue: true/false` のみ返却
- ガードレール違反時のガイダンスが限定的

**対応内容**:
- [x] `pretooluse-guard.sh` を拡張し、コンテキスト別のガイダンス返却
- [x] スキル実行中の品質ルール注入（test-quality.md, implementation-quality.md）
- [ ] ファイル編集時のパターン推奨（patterns.md から関連パターン抽出）

**実装例**:
```json
{
  "continue": true,
  "additionalContext": "【ガイドライン】このファイルを編集する際は...\n- TDD を優先\n- 既存テストの改ざん禁止"
}
```

**テストケース**:
| 入力（ツール/ファイル） | 期待出力 | 備考 |
|------------------------|---------|------|
| Edit tests/*.spec.ts | テスト改ざん禁止ガイド注入 | test-quality.md 適用 |
| Write src/services/*.ts | 実装品質ガイド注入 | implementation-quality.md 適用 |
| Bash npm test | ガイドなし（pass-through） | 読み取り系は軽量 |

### 33.3 SessionStart agent_type 対応 `cc:完了`

> **v2.1.2 新機能**: SessionStart フック入力に `agent_type` が追加

**現状**:
- `session-init` は全セッションで同一初期化
- サブエージェントとメインエージェントの区別なし

**対応内容**:
- [x] `session-init.sh` で agent_type を判定
- [x] メインエージェント: フル初期化（Plans.md 状態表示、claude-mem コンテキスト）
- [x] サブエージェント: 軽量初期化（タスク固有情報のみ）
- [ ] task-worker 起動時の初期化最適化

**実装例**:
```bash
# session-init.sh
if [ "$AGENT_TYPE" = "subagent" ]; then
  # 軽量初期化
  echo '{"hookSpecificOutput":{"minimal":true}}'
else
  # フル初期化
  ...
fi
```

  - `session-summary`: 30秒 → 60秒（大規模セッション）
