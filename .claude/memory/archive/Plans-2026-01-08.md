## フェーズ15: `/harness-init` 対話効率化 `cc:TODO`

> **目的**: 初期化工程の対話回数を削減し、ユーザー体験を向上（質は維持）
> **Before**: 最大11回の対話ラウンド → **After**: 最大2回

### Phase 1: 対話回数削減の設計 `cc:TODO`

#### 現状の問題点

| フェーズ | 待機回数 | 質問内容 |
|---------|---------|---------|
| Phase 0 | 3回 | 言語選択 → モード選択 → 2-Agent確認 |
| Phase 1.5 | 1回 | 曖昧ケース確認 |
| Phase 2A | 4回 | やりたいこと → 解像度 → 技術スタック → プロジェクト名 |
| Phase 3 | 1回 | Skills Gate設定 |
| Phase 4 | 1回 | 不足ファイル確認 |
| **合計** | **最大11回** | |

#### 改善後のフロー

```
Step 1: 統合質問（1回）
  ├─ 何を作る？（または既存プロジェクトへの導入）
  ├─ 対象ユーザーは？（自分/チーム/一般公開）
  └─ おまかせ or 詳細設定？

Step 2: 確認（おまかせなら省略）
  └─ 技術スタック + プロジェクト名

→ セットアップ実行（バックグラウンドで分析）

Step 3: 完了報告（待機なし）
───────────────────────────
最小 1回、最大 2回の対話ラウンド
```

## 使い方

1. **タスク追加**: `/plan-with-agent` で新しいタスクを追加
2. **タスク実行**: `/work` でタスクを実装（マーカーが自動更新）
3. **レビュー**: `/harness-review` で品質チェック
4. **整理**: `/cleanup` で完了タスクをアーカイブ
